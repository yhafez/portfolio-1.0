Dwellingly (In Progress)




Dwellingly is a volunteer civic-coding project that I am working on with Code for PDX. We are in the process of building an administrative dashboard that will be utilized across organizations in the Portland non-profit sector with supportive-housing programs. The goal of the project is to implement a solution in place of the current dated infrastructure to facilitate communication between recently-homeless tenants transitioned into supportive housing, the social workers they are assigned to, and the landlords of the properties they are placed in.


This project is a fullstack web application built with React, Bulma, and SASS for the web interface, and Python and SQLite for the database and back-end, with Jest for front-end testing. As a contributing developer, I have been responsible for building user-facing React components and integrating them into a code-base with over 40 volunteer contributors. My involvement in this project has helped me grow tremendously in my understanding of Git/Github and how they're used in industry best-practices. Aside from getting to learn and practicing new technologies, the project also meaningfully exposed me to what it's like to work in a real-world product team. I've also had the opportunity to contribute to onboarding and training new members, and have had many opportunities to mentor others on my own knowledge of web development and React.

The Great Linkerator

github


A collaborative initiative between myself and the talented Tony Dyleuth. Working with these libraries has pushed my design and styling abilities to a new level. Juice-Box

github

"Juicebox" is my first back-end only web application and is a REST API simplistically mimicing the functionality of Tumblr. Newzy


The "Newzy" project is a search aggregator web application which allows users to do basic and advanced searches for news articles from across the web which are returned in reverse chronological order. Users can also see recent news not limited by search criteria by leaving the search bar empty and clicking search, specify the language and region which they would like to limit their search results to in the settings, and view related searches by clicking on the tags in a search result item.


This project involved more practice with AJAX and fetch calls, query strings, date formatting for API calls, API keys, and tokens, and utilitizing the Currents API. I especially enjoyed flexing my creativity in the styling for this project and coming up with a witty name and description to go with the concept 😌. In a small coincidence during an office hour session, one of my instructors downloaded my code files to run locally on his machine and, to my horror, the title logo, which I created myself as a transparent PNG image using my graphic design skills in order to add a more complex shadow and use a non-web friendly font (and to add some spice to my website using skills I picked up before learning web development 💁‍♀️), completely disappeared! This was because it used a relative path to point to the image source, and since the website was not yet deployed with all its resources, downloading the code files locally instead made it so the code pointed to a local source which was non-existent on my instructors device. The silver lining in that experience was that it taught me about using JavaScript event handlers to listen for errors in image rendering to then substitute the automatic "broken image" notice which appears when an image fails to properly load with plain text fancily styled in CSS ahead of time in case of emergencies just like this one! Including error handling for images to enable my applications to more gracefully deprecate became a regular practice of mine moving forward.
